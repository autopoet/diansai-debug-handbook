# 电赛白皮书全栈学习路线

> 目标：完成可公开演示的产品，同时具备应聘全栈或后端实习所需的项目理解

## 学习方式

每个阶段都完成四件事：

1. 能用自己的话解释原理。
2. 独立完成一个最小练习。
3. 将知识应用到项目功能。
4. 写测试并能解释实现取舍。

不要把“代码能够运行”等同于“已经掌握”。每个阶段结束时，应能不看代码画出请求流程或数据关系。

## 阶段 0：开发环境与 Git

学习内容：

- 前后端目录结构。
- Git 工作区、暂存区、提交和远程仓库。
- 环境变量、依赖和 `.gitignore`。

项目产出：

- React 和 FastAPI 均可启动。
- `/api/v1/health` 可以返回健康状态。
- 前端可以请求并显示后端状态。

面试表达：

- 为什么前后端分离？
- 为什么凭据不能提交到 Git？
- 一次 Git 提交应如何保持清晰？

## 阶段 1：HTTP 与 FastAPI 基础

学习内容：

- HTTP 方法、URL、请求头、请求体和状态码。
- FastAPI 路由、依赖、Pydantic 校验和异常响应。
- 同步与异步的基本区别。

项目产出：

- 故障现象和条目的只读 API。
- 分页、筛选和统一错误结构。
- API 自动文档可用。

面试表达：

- `GET`、`POST`、`PUT`、`PATCH` 有什么区别？
- `401`、`403`、`404`、`409` 分别表示什么？
- 为什么同步 Peewee 不能直接阻塞异步事件循环？

## 阶段 2：数据库与 Peewee

学习内容：

- 表、行、主键、外键、唯一约束和索引。
- 一对多、多对多关系。
- Peewee Model、查询、事务和连接生命周期。
- SQLite 与 PostgreSQL 的区别。

项目产出：

- 用户、条目、版本、标签和分类模型。
- 数据库迁移。
- 种子数据和数据库测试。

面试表达：

- ORM 解决什么问题，又有什么代价？
- 索引为什么能加速查询，为什么不能滥用？
- 什么是事务和 ACID？

## 阶段 3：注册、登录与权限

学习内容：

- 密码哈希、会话、Cookie、Token。
- 认证与授权的区别。
- RBAC 角色权限模型。
- CSRF、XSS、暴力破解和接口限流。

项目产出：

- 用户名/邮箱/密码注册登录。
- 普通用户、审核员和管理员权限。
- 登录、退出和权限测试。

面试表达：

- 为什么不能直接保存密码？
- JWT 和服务端 Session 各有什么取舍？
- 如何阻止普通用户访问审核接口？

## 阶段 4：条目版本与审核

学习内容：

- 不可变版本快照。
- 状态机和业务不变量。
- 乐观并发控制。
- 跨表事务。

项目产出：

- 提交待审核版本。
- 审核差异、通过、驳回和重新提交。
- 审核期间保持旧版本公开。
- `409 Conflict` 处理。

面试表达：

- 为什么 Article 和 ArticleRevision 要分开？
- 如何避免两个人的修改互相覆盖？
- 审核通过为什么必须使用事务？

## 阶段 5：富文本与划线评论

学习内容：

- ProseMirror 结构化文档模型。
- 浏览器 Selection 和 Range。
- 文本引用、上下文与位置锚点。
- 文档更新后的评论重定位。

项目产出：

- 富文本编辑与结构化 JSON。
- 选中文字评论、回复和解决。
- 评论脱锚后的安全降级显示。

面试表达：

- 为什么正文不只保存 HTML？
- 正文修改后如何避免评论绑定到错误文字？
- 实时协同为什么不适合首个版本？

## 阶段 6：搜索、图片与性能

学习内容：

- PostgreSQL 全文检索。
- 文件类型与大小校验。
- 对象存储。
- 分页、缓存、慢查询和 N+1 问题。

项目产出：

- 按故障现象搜索和筛选。
- 波形图片上传。
- 查询性能测量和索引优化记录。

面试表达：

- 数据库全文检索与搜索引擎如何选择？
- 为什么文件扩展名不能代表真实文件类型？
- 如何定位一个慢接口？

## 阶段 7：测试、部署与可观测性

学习内容：

- 测试金字塔。
- CI、环境配置和数据库迁移。
- HTTPS、反向代理、日志、监控和备份。

项目产出：

- 核心流程自动化测试。
- GitHub Actions 持续集成。
- 可公开访问的演示环境。
- README 架构说明、演示截图和部署文档。

面试表达：

- 如何安全发布数据库结构变化？
- 如何发现和定位线上错误？
- 数据库备份为什么必须做恢复演练？

## 最终求职材料

项目完成后准备：

- 一条在线演示地址。
- 一份清晰的 GitHub README。
- 一张系统架构图和一张核心数据模型图。
- 3 分钟项目介绍。
- 版本审核、并发冲突、评论锚点三个技术难点的深度讲解。
- 性能测试、自动化测试和线上部署的真实数据。
- 对当前不足和下一步优化的诚实总结。
