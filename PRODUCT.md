# Product

## Register

product

## Platform

web

## Users

主要用户是处在比赛准备、实验室联调或赛场排障环境中的电赛参赛学生。他们通常时间紧张，已经观察到某种故障现象，希望尽快获得可靠、可执行、可验证的排查路径。

有经验的往届选手和队员是主要贡献者，负责沉淀和修正实践经验。指导教师与领域专家负责补充依据、识别风险和提高结论可信度。项目负责人在 MVP 阶段同时承担审核员和管理员职责。

## Product Purpose

电赛白皮书将分散在聊天记录、个人笔记、博客和口头交流中的 Debug 经验，整理为“故障现象 → 快速检查 → 分步排查 → 根因 → 解决方案 → 修复验证”的公开协作知识库。

产品需要让读者快速找到并执行排查步骤，也让注册用户能够通过富文本编辑、版本提交、审核和划线讨论持续改进内容。成功意味着用户能更快定位故障，且知识结论可追溯、可复现、可验证。

## Positioning

从可观察的故障现象出发，提供面向电赛现场、经过审核且可以逐步执行的 Debug 知识。

## Brand Personality

专业、冷静、可信。界面应让焦虑且专注的排障者迅速进入任务，既有飞书文档的协作效率、Linear 的交互精度，也有 Wikipedia 的信息可信感。

## Anti-references

不做以宣传为中心的营销官网，不使用通用 SaaS 功能卡片墙，也不采用电竞风、赛博朋克、玻璃拟态、装饰性渐变或无意义的动态效果。技术感来自结构、排版、数据和操作精度，而不是视觉噱头。

## Design Principles

- 搜索先于介绍：第一屏首先帮助用户描述故障和找到答案。
- 现象先于学科：以用户观察到的问题组织入口，技术方向只作辅助筛选。
- 正文先于界面：阅读页保持专注，目录、评论和元信息按需出现。
- 操作必须可追溯：提交、审核、版本差异和驳回原因始终清晰可见。
- 高风险内容诚实醒目：适用条件、安全警告、验证状态和过期信息不能被装饰弱化。

## Accessibility & Inclusion

以 WCAG AA 为基础目标。核心阅读和操作支持键盘访问、可见焦点与减少动画；图片提供替代文本；状态不能只依赖颜色表达。阅读体验适配手机，编辑和审核在 MVP 阶段以桌面端为主。
