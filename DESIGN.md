# 电赛白皮书设计系统

## 设计方向

以“技术编辑部”作为整体骨架，吸收协作文档的编辑、版本与评论交互。界面服务于搜索、阅读和贡献，视觉重点始终是知识内容，而不是装饰或产品宣传。

使用场景是明亮实验室中的专注排障：浅色优先、结构清楚、信息可信。文章保持普通富文本形态，不要求贡献者使用复杂流程组件。

## 色彩

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| Canvas | `#F7FAF8` | 全站背景 |
| Surface | `#FFFFFF` | 文档、浮层和控件表面 |
| Ink | `#16211B` | 标题与主要正文 |
| Muted | `#5F6D65` | 元信息与辅助说明 |
| Border | `#DCE5DF` | 分隔线和控件边界 |
| Control Border | `#829188` | 输入框等必要交互边界 |
| Signal Green | `#187A50` | 主操作、当前状态和焦点 |
| Soft Green | `#E8F4ED` | 选中、通过和轻量强调 |
| Warning | `#A45C16` | 风险提示 |
| Danger | `#B42318` | 错误和危险状态 |

Signal Green 是唯一品牌强调色，只用于操作和状态，不作大面积装饰。警告和危险色仅表达语义。

## 字体

- 中文和界面：`Noto Sans SC`，回退至 `PingFang SC`、`Microsoft YaHei` 和系统无衬线字体。
- 代码与测量值：`JetBrains Mono`，回退至 `SFMono-Regular`、`Consolas` 和系统等宽字体。
- 固定字号阶梯：`0.75rem / 0.875rem / 1rem / 1.25rem / 1.5rem / 2rem`。
- 正文至少 `1rem`，文章正文行高 `1.75`，舒适行宽 `65–72ch`。
- 中文标题字距不小于 `-0.02em`，不使用营销页面式流体巨型标题。

## 间距与形状

- 4pt 基础尺度：`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64 / 96px`。
- 控件圆角 `6px`，面板圆角 `10px`，标签可使用全圆角。
- 正文主要依赖留白和分隔线组织；仅浮层使用轻阴影。
- 不使用“边框 + 大范围柔光阴影”的幽灵卡片组合。

## 页面布局

- 全站顶栏为全宽边界，内容在内部容器中对齐。
- 首页与探索页最大内容宽度约 `1120px`。
- 阅读页宽屏采用左侧目录、居中文档和按需评论抽屉；正文不因评论打开而重排。
- 文章正文约 `680–720px`，目录约 `200–224px`。
- 小屏折叠目录，评论使用全高抽屉；表格和代码只在自身容器横向滚动。

## 组件原则

- 搜索是首页第一操作，输入两个字符后才请求建议。
- 搜索结果使用带分隔线的文本列表，不使用同款卡片墙。
- 文档内容使用标准标题、段落、列表、表格、代码块、图片与提示块。
- 评论默认收起，选中文字或激活评论入口后再展开。
- 加载使用稳定尺寸的骨架；空状态说明下一步；错误状态保留用户的查询并提供重试。

## 动效

- 交互反馈使用 `150–200ms` 的 ease-out 过渡。
- 动效只表达搜索建议出现、导航状态和抽屉开合，不做编排式入场动画。
- `prefers-reduced-motion` 下取消位移、平滑滚动和非必要过渡。

## 可访问性

- 目标为 WCAG AA，正文与背景对比度至少 `4.5:1`。
- 所有交互提供 `:focus-visible` 焦点环，触控目标至少 `44×44px`。
- 使用正确的页面地标、标题层级、跳转主内容链接和异步状态播报。
- 状态不能只靠颜色表达；图片和波形必须有替代说明。
- 页面支持键盘操作、200% 文本缩放以及 `375 / 768 / 1024 / 1440px` 宽度。
